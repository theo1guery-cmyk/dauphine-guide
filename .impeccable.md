## Design Context

### Users
Étudiants en L1 Droit-Éco à Paris-Dauphine PSL, 18–20 ans. Ambitieux, connectés, exigeants. Utilisent l'app depuis leur téléphone entre deux cours, chez eux pour réviser, ou en amphi pour consulter leur emploi du temps. Contexte : bureau bien éclairé, bibliothèque, chambre le soir. Cherchent à réussir leur L1 avec les meilleurs outils disponibles.

### Brand Personality
**Jeune · Ambitieux · Élégant**

Voix : directe, énergique, jamais condescendante. Ton d'un guide premium écrit par un pair qui a déjà réussi. L'interface doit dégager confiance et excellence — comme si Linear et Notion s'étaient améliorés pour des étudiants de grande école.

Émotions cibles : confiance, motivation, appartenance à une communauté d'élite, plaisir d'utiliser quelque chose de vraiment bien fait.

### Aesthetic Direction
**Primaire : Light mode élégant** avec un accent bleu institutionnel Dauphine (`#003d7a`) enrichi. Le dark mode est un complément raffiné, pas une réflexion après coup.

- Palette : bleu profond → bleu vif → blanc chaud, avec des neutres légèrement teintés vers le bleu
- Typographie : combinaison d'un serif expressif pour les headlines + un sans-serif précis pour l'UI
- Mouvement : **spectaculaire, fluide, vivant** — animations d'entrée dramatiques, micro-interactions réactives, transitions de page lisses
- Anti-références : pas institutionnel/gouvernemental, pas plat/ennuyeux, pas générique SaaS-bleu-gris

### Design Principles
1. **Alive by default** — chaque surface doit donner l'impression que l'app respire. Transitions fluides, particules subtiles, états hover expressifs.
2. **Élégance, pas ostentation** — les effets servent la clarté. Un shimmer sur une carte, pas un feu d'artifice.
3. **Hiérarchie dramatique** — gros contraste de taille entre les niveaux typographiques. Les headlines commandent l'attention.
4. **Premium à chaque pixel** — espacement précis, arrondis cohérents, ombres qui ont du sens. Jamais de générique.
5. **Mobile-first ambition** — l'app est souvent utilisée sur téléphone : les interactions tactiles doivent être aussi satisfaisantes que les hover desktop.
